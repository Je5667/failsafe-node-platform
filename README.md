# Fail-Safe OTA 기반 분산 제어 노드 자가복구 플랫폼

본 프로젝트는 임베디드 SW 공모전을 목표로 진행되는 플랫폼 개발 프로젝트입니다. CAN 기반 분산 제어 시스템에서 장애를 감지하고 자가 복구(Self-Healing)를 수행하는 기반 아키텍처를 설계하고 구현하는 것을 목표로 합니다.

## 프로젝트 소개

### 목적

이 프로젝트는 특정 응용 분야에 국한되지 않고, 분산 제어 노드 간의 통신과 장애 대응 기능을 포함하는 공통 플랫폼을 개발하는 데 중점을 둡니다. 현재 Repository는 실제 애플리케이션보다 "플랫폼" 자체를 구축하는 단계입니다.

### 왜 분산 제어 시스템이 필요한가

- 여러 노드가 함께 동작하는 환경에서는 단일 MCU 기반 구조로는 장애 확산과 중앙 집중식 실패에 취약합니다.
- 분산 제어 시스템은 각 노드가 독립적으로 상태를 모니터링하고, 네트워크 기반으로 협력하여 전체 시스템 안정성을 높입니다.

### 왜 Fail-Safe와 Self-Healing이 중요한가

- 장애가 발생하더라도 시스템이 안전 모드로 진입하고, 가능한 경우 정상 상태로 복귀해야 합니다.
- 자가 복구는 사람의 개입 없이도 노드의 장애를 탐지하고 격리하며 복구 절차를 수행하는 능력입니다.
- 이런 기능은 산업용, 인프라, 안전 제어 시스템에서 신뢰도를 크게 향상합니다.

### 기존 단일 MCU 구조의 한계

- 단일 MCU 기반 설계는 확장성과 유연성이 낮습니다.
- 장애 발생 시 전체 서비스가 중단될 수 있으며, 장애 위치를 분리하기 어렵습니다.
- 하드웨어 종속성과 특정 통신 인터페이스에 대한 결합도가 높아지고, 확장 시 많은 수정이 필요합니다.

## 프로젝트 목표

본 플랫폼은 다음 기능을 중심으로 설계됩니다.

- CAN 기반 노드 간 통신
- Heartbeat 기반 상태 모니터링
- 노드 장애 감지
- 장애 노드 격리
- 자가 복구(Self-Healing) 메커니즘
- OTA 업데이트 지원
- 확장 가능한 통신 구조

이 구조는 CAN을 기본으로 하되, 상위 계층에서는 통신 방식에 종속되지 않도록 설계되어 향후 Ethernet/UART로 확장하기 쉽습니다.

## 시스템 구조

```
Application Layer
Middleware Layer
Hardware Abstraction Layer
STM32 HAL Driver
CAN Hardware
```

이 계층 구조는 상위 로직과 하드웨어 구현을 명확히 분리합니다.

- Application Layer: 시스템 동작을 결정하고 서비스 흐름을 제어합니다.
- Middleware Layer: 상태 관리, 프로토콜 처리, Heartbeat 및 Fault Manager와 같은 핵심 기능을 제공합니다.
- Hardware Abstraction Layer: CAN, Timer, OTA 등 하드웨어 의존 기능을 추상 인터페이스로 감쌉니다.
- STM32 HAL Driver: 실제 STM32 HAL 호출을 포함하는 하위 구현부(추후 연동).
- CAN Hardware: 물리적 CAN 네트워크와 통신하는 최하위 계층입니다.

### Wrapper 구조를 사용하는 이유

- 상위 계층은 특정 MCU나 HAL API에 직접 의존하지 않습니다.
- 통신 인터페이스 변경 시 상위 로직을 수정하지 않고 하위 래퍼만 교체할 수 있습니다.
- 단위 테스트와 시뮬레이션 개발을 위해 하드웨어 의존성을 분리할 수 있습니다.

## Repository 구조

- `Application/` : 애플리케이션 진입점과 실행 흐름을 정의합니다.
- `Middleware/` : 노드 상태, Heartbeat, Fault Management, 프로토콜 정의 등 플랫폼 로직을 포함합니다.
- `Hardware/` : CAN, Timer, OTA에 대한 추상화 래퍼 구현을 포함하며, 실제 하드웨어 연동을 분리합니다.
- `docs/` : 아키텍처, 프로토콜, 상태 기계 등 설계 문서입니다.
- `tests/` : 플랫폼 기능 검증을 위한 테스트 코드 및 검증 스텁입니다.

## 적용 가능 분야

본 플랫폼은 특정 프로젝트가 아닌 다양한 산업 시스템에 공통으로 적용 가능한 기반 플랫폼입니다.

- 전기차 충전소 분산 제어 시스템: 충전기 노드 간 상태 동기화와 장애 격리
- 산불 감시 및 대응 시스템: 센서 노드 장애 감지와 네트워크 기반 예비 동작
- 스마트팜 환경 제어 시스템: 온도/습도 제어 노드의 자율 복구
- 스마트 팩토리 설비 제어 시스템: 생산 설비 노드의 상태 모니터링과 장애 대응
- 산업용 IoT 네트워크: 분산 노드의 신뢰성 확보와 원격 펌웨어 업데이트

각 분야에서 본 플랫폼은 분산 제어, 장애 감시, 복구 로직을 공통으로 제공하여 적용 범위를 넓힐 수 있습니다.

## 개발 로드맵

- Phase 1: 리포지토리 구조 설계 및 프로토콜 정의
- Phase 2: Heartbeat 및 상태 관리 기초 구현
- Phase 3: Fault Detection 및 장애 상태 처리
- Phase 4: STM32 CAN 통신 연동
- Phase 5: OTA 업데이트 기능 구현
- Phase 6: 자가 복구(Self-Healing) 기능 완성

## 기대 효과

- 시스템 신뢰성 향상: 분산 노드 간 협업으로 장애 영향을 줄입니다.
- 유지보수 비용 감소: 장애 감지와 복구를 자동화하여 관리 부담을 낮춥니다.
- 장애 대응 시간 단축: 빠른 상태 전환과 격리를 통해 시스템 안정성을 확보합니다.
- 확장성 확보: 하드웨어 추상화를 통해 향후 통신 인터페이스 확장이 용이합니다.

## 주의사항

본 Repository는 현재 플랫폼 아키텍처 설계 및 초기 구현 단계입니다. STM32CubeMX 연동과 실제 하드웨어 통합은 이후 단계에서 진행할 예정입니다.
