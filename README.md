# 🛡️ Fail-Safe OTA 기반 분산 제어 노드 자가복구 플랫폼
> CAN 기반 분산 제어 시스템에서 장애를 감지하고 자가 복구(Self-Healing)를 수행하는 임베디드 플랫폼

| 항목 | 내용 |
| --- | --- |
| 목표 | 분산 제어 노드의 장애 감지 및 복구를 위한 플랫폼 설계 |
| MCU | STM32 기반 아키텍처 방향 |
| 통신 | CAN 기반 메시지 교환 |
| 핵심 기능 | Heartbeat · Fault Detection · Self-Healing · OTA |

---

# 📖 프로젝트 소개

## 프로젝트 목적

임베디드 SW 공모전을 목표로 하는 플랫폼 개발 프로젝트입니다. 이 Repository는 특정 응용보다 분산 제어 플랫폼의 구조와 확장성에 초점을 맞춥니다.

## 왜 분산 제어 시스템인가

- 여러 노드가 협업하여 전체 시스템 동작을 분산 처리합니다.
- 중앙 집중식 장애점(SPOF)을 줄이고 가용성을 높입니다.
- 확장 시 추가 노드가 유연하게 결합될 수 있습니다.

## 기존 단일 MCU 구조의 한계

- 단일 MCU는 장애 시 전체 서비스 정지 위험이 큽니다.
- 하드웨어, 통신 인터페이스에 대한 결합도가 높습니다.
- 분산 환경에서 상태 관리와 장애 대응이 부족합니다.

## 왜 Fail-Safe와 Self-Healing이 중요한가

- 장애 발생 시 안전 모드로 전환하고 잠재적 위험을 줄입니다.
- 노드 단위로 장애를 탐지하고 복구 절차를 자동화합니다.
- 운영 환경에서 유지보수 부담을 줄이고 서비스 연속성을 확보합니다.

# 🎯 프로젝트 목표

- **CAN 기반 노드 간 통신**  
  노드 간 제어 신호와 상태 정보를 CAN 메시지로 교환합니다.

- **Heartbeat 기반 상태 모니터링**  
  주기적 생존 신호를 통해 통신 이상과 노드 상태를 감시합니다.

- **노드 장애 감지**  
  통신 지연, 메시지 손실, 상태 불일치를 감지하여 장애를 식별합니다.

- **장애 노드 격리**  
  문제가 있는 노드를 분리하여 정상 노드 운영을 유지합니다.

- **자가 복구 메커니즘**  
  장애 상태에서 자동 복구 흐름을 구축하고 재진입을 지원합니다.

- **OTA 업데이트**  
  원격 펌웨어 업데이트를 위한 제어 프레임워크를 준비합니다.

- **확장 가능한 통신 구조**  
  상위 계층은 통신 방식에 의존하지 않도록 설계하여 확장을 용이하게 합니다.

# 🏗️ 시스템 구조

```text
Application Layer
Middleware Layer
Hardware Abstraction Layer
STM32 HAL Driver
CAN Hardware
```

## Wrapper 구조를 사용하는 이유

- 하드웨어 의존성 감소
- 플랫폼 이식성 향상
- 테스트 용이성 확보

상위 계층은 STM32 HAL API를 직접 호출하지 않고, 추상화된 래퍼를 통해 하드웨어와 통신합니다.

# 🚀 적용 가능 분야

이 플랫폼은 특정 제품이 아니라 다양한 산업 시스템에 공통으로 적용 가능한 기반 구조를 지향합니다.

| 분야 | 활용 예시 |
| --- | --- |
| 전기차 충전소 분산 제어 | 충전기 노드 상태 동기화 및 장애 격리 |
